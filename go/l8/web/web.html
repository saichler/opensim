Layer 8 Ecosystem

Simulated infrastructure for operator workflows, not demo screenshots.

OpenSim creates realistic network and GPU device fleets with SNMP v2c/v3, SSH, VT100 terminal behavior, HTTPS APIs, and dynamic metrics so monitoring, automation, and inventory tooling can be exercised against something closer to production.

Start with OpenSim Open the app

What it supports

  • Large simulated fleets with realistic interface, protocol, and metric behavior.
  • Network and GPU profiles for monitoring systems, collectors, and automation flows.
  • A web app, API surface, CSV export, and route generation for repeatable test environments.

Overview

A simulator built for coverage, scale, and repeatability.

OpenSim is designed to help teams validate monitoring, discovery, polling, and automation behavior before those systems are pointed at live infrastructure.

Concurrency

25,000+

Concurrent devices when you need large-scale monitoring and load scenarios.

Coverage

28

Device types across routing, switching, storage, servers, and GPU infrastructure.

Delivery

100%

Source-available workflows that can be inspected, extended, and embedded into test environments.

Components

The system combines transport realism, device profiles, and operational control.

Instead of a generic simulator shell, OpenSim brings together the interfaces, protocol behavior, and operator controls needed to exercise real workflows.

Protocols

SNMP, SSH, and HTTPS surfaces

Test collectors and automation against SNMP v2c/v3, SSH terminal sessions with VT100 behavior, and HTTPS-backed APIs without stitching together multiple tools.

Scale

Large fleets with controlled resource use

Build high-count environments with namespace isolation, pre-generated metrics, and fleet creation flows designed for repeatable large-scale test runs.

Profiles

Network, storage, and GPU device coverage

Simulate routers, switches, storage systems, and NVIDIA platforms including DGX and HGX variants, each with profile-specific metrics and behavior.

Metrics

Dynamic state instead of static mock values

Drive CPU, memory, temperature, power, and GPU utilization changes so downstream systems can be tested against moving telemetry rather than fixed fixtures.

Operations

Interactive fleet management

Create, filter, inspect, export, and route to device fleets through the web app and API without rebuilding the environment for every scenario.

Workflows

Designed for validation, not presentation

Use OpenSim for discovery testing, polling validation, SSH automation, performance runs, and CI paths that need repeatable infrastructure states.

Use Cases

Where teams use OpenSim

OpenSim is most useful where infrastructure software needs broad device coverage and controlled, repeatable environments.

Network monitoring validation

Exercise polling pipelines, topology discovery, and alert logic against realistic SNMP device fleets.

GPU infrastructure testing

Validate GPU observability tools against simulated DGX and HGX telemetry and inventory data.

SSH automation development

Build and verify interactive automation against terminal behavior, routing context, and device prompts.

Load and scale assessment

Understand how collectors, APIs, and workflow engines behave when the simulated fleet size increases sharply.

CI and regression environments

Run repeatable infrastructure tests in pipelines without depending on scarce or changing hardware labs.

Operator training and demos

Provide realistic environments for product walkthroughs and operator onboarding without production risk.

Start

From clone to first fleet

The fastest path is to build the simulator locally, run it with the privileges needed for TUN/TAP interfaces, and create devices through the API or web app.

1

Clone and build

Build the simulator binary from the repository.

git clone https://github.com/saichler/opensim.git cd opensim/go/simulator go build -o simulator .

2

Run the simulator

Start the process with the permissions required to create interfaces.

sudo ./simulator

3

Create your first set

Provision a small test fleet through the API.

curl -X POST http://localhost:8080/api/v1/devices \ -H "Content-Type: application/json" \ -d '{"start_ip":"192.168.100.1","device_count":5,"netmask":"24"}'

Application

The app shell is built for provisioning and inspection.

The OpenSim web app gives operators one place to create fleets, review status, filter devices, export inventory, and generate route scripts for local host access.

  • Provision fleets by start IP, count, category, and device profile.
  • Review system memory, CPU usage, load, and fleet totals in one shell.
  • Export CSV output and generate permanent route scripts directly from the app.
Launch the app
localhost:8080/ui
